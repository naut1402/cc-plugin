Dev Agents Team — sơ đồ tương tác

Click vào dev-team-orchestrator để xem nó gọi 6 agent theo thứ tự (đường teal có số). Click một agent để xem skill/artifact nó dùng (đường cam đứt nét). Click HITL-doc để xem vòng lặp review tài liệu — conditional, do user opt-in tại mỗi gate. Q&A là inline checkpoint phát sinh bất ngờ trong 3 phase. Click lại ô đang chọn để bỏ chọn.

Model nhỏ dưới tên agent: sonnet / opus / haiku. ✦ = HITL-doc chỉ xuất hiện khi user opt-in tại gate sau HITL #1 hoặc #2. ⟳ = Q&A inline có thể phát sinh bất ngờ trong investigator / designer / implementer. Đường cam đứt nét = uses/writes/reads · Đường teal liền = dev-team-orchestrator gọi agent (1–6).
